Huffman File Compressor Project

This project implements a Huffman File Compressor using the C++ programming language. The objective of this project is to reduce the storage size of files without losing any information. Huffman Coding is a well-known lossless data compression algorithm that assigns shorter binary codes to frequently occurring characters and longer binary codes to less frequent characters. This approach reduces the average number of bits required to represent data and therefore decreases the overall file size.

The compressor begins by reading the input file in binary mode. Every byte of the file is processed, and a frequency table is generated. The frequency table contains the number of occurrences of each possible byte value from zero to two hundred and fifty-five. Characters that appear more frequently are considered more important during the construction of the Huffman Tree. The generated frequency table becomes the foundation of the compression process.

After calculating the frequency of every byte, the program constructs the Huffman Tree. A priority queue is used to efficiently select the two nodes with the smallest frequencies. These two nodes are combined into a new parent node whose frequency is equal to the sum of both child nodes. This process continues until only one node remains in the priority queue. The remaining node becomes the root of the Huffman Tree. Every leaf node in the tree represents a unique byte from the original file.

Once the Huffman Tree has been created, the compressor traverses the tree to generate Huffman Codes. Moving to the left child appends a zero to the current binary code, while moving to the right child appends a one. When a leaf node is reached, the generated binary sequence becomes the Huffman Code for that particular byte. Because frequently occurring bytes are closer to the root of the tree, they receive shorter binary codes. Less common bytes receive longer binary codes. This property makes Huffman Coding an efficient compression technique.

The original file is then read again. Instead of storing every byte directly, each byte is replaced with its corresponding Huffman Code. This produces a long sequence of binary digits. Since computers store information as bytes, the binary digits are packed into groups of eight bits using bitwise operations. These packed bytes become the compressed data that is written to the output file. If the total number of bits is not a multiple of eight, extra padding bits are added to complete the final byte.

The compressed file contains more than just compressed data. It also stores metadata that allows the original file to be reconstructed exactly. The metadata includes the number of unique characters, the frequency table for those characters, and the number of padding bits added during compression. Without this information, the decompressor would not be able to rebuild the Huffman Tree correctly. The metadata is written before the compressed binary stream so that it can be read during decompression.

During decompression, the compressed file is opened in binary mode. The metadata is read first, and the stored frequency table is reconstructed. Using this frequency table, the exact same Huffman Tree is built again. The compressed binary stream is then processed one bit at a time. Every zero causes the traversal to move to the left child of the Huffman Tree, while every one causes the traversal to move to the right child. Whenever a leaf node is reached, the original byte is recovered and written to the output file. This process continues until every valid bit has been processed.

One of the major advantages of Huffman Coding is that it is completely lossless. Lossless compression means that every bit of the original data can be recovered perfectly after decompression. Unlike lossy compression techniques, Huffman Coding never removes information from the file. Therefore, text documents, source code, executable files, configuration files, spreadsheets, and many other file types can be restored exactly as they were before compression.

Although Huffman Coding works on every type of file, the amount of compression depends on the characteristics of the input data. Plain text files often contain many repeated characters and repeated words, making them excellent candidates for Huffman Coding. Files such as PDF documents, JPEG images, PNG images, ZIP archives, MP4 videos, and other already compressed formats usually show little improvement because they have already been optimized using other compression algorithms. In some situations, the compressed output may even become slightly larger because the metadata itself occupies additional space.

This project demonstrates several important concepts of data structures and algorithms. It makes use of priority queues, binary trees, recursion, unordered maps, bitwise operations, binary file input and output, object-oriented programming, and memory management. These concepts are commonly discussed during software engineering interviews and provide a strong foundation for understanding file compression systems.

The project also includes a web interface developed using the Flask framework. The web application allows users to upload a supported file, choose either compression or decompression, and download the processed output through a simple graphical interface. Temporary files are stored only during processing and are automatically cleaned up after download. The application provides information such as the original file size, processed file size, and compression ratio. A sample text file is also available so that users can quickly test the application without creating their own input data.

Overall, this Huffman File Compressor demonstrates how theoretical algorithms can be transformed into practical software. The project combines efficient algorithm design, binary file handling, object-oriented programming, and web development into a single application. It provides a complete workflow starting from reading an input file, constructing a Huffman Tree, generating binary codes, compressing data, storing metadata, rebuilding the Huffman Tree, decoding the compressed bit stream, and finally recovering the original file without any loss of information. This project highlights the importance of algorithms, data structures, and software engineering principles in solving real-world problems while providing an efficient and user-friendly compression tool.
